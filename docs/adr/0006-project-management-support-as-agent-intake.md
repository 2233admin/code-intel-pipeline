# Treat project management support as agent intake

Accepted. Code Intel Pipeline internalizes the `mattpocock/skills` setup concepts of issue tracker selection, triage label vocabulary, and domain docs as project-management support for agents. Linear is the preferred external issue tracker for this repo, and Obsidian/LLM wiki workflows are supported as optional knowledge surfaces.

This remains agent intake, not scanner runtime dependency. Code Intel Pipeline does not require `mattpocock/skills`, Linear clients, Obsidian plugins, wiki tooling, or external credentials to produce artifact runs. Scanner artifacts remain authoritative evidence; Linear issues and wiki notes may reference or summarize them but must not replace artifact data contracts, verification evidence, or safety boundaries.
