# Keep Code Evidence Layer v1 hybrid and avoid full call graph

Accepted. Code Evidence Layer v1 will produce full machine-readable dumps for files, symbols, chunks, symbol-to-chunk mappings, and basic imports, plus curated Agent Code Slices for default Agent consumption. It will not promise a complete cross-language call graph in v1; call relationships may appear only as experimental candidates until their precision, recall, and framework behavior can be measured. The v1 implementation must prove system connectivity end to end: evaluation harness output must be written into the artifact run, surfaced through `understanding.md` as the Agent handoff entry point, linked from `summary.md` as status, and reported as a typed optional outcome rather than an isolated benchmark. A minimal native Code Evidence Layer is enabled by default so every artifact run can expose an Agent Code Slice; `cocoindex-code` remains an optional adapter that skips cleanly unless explicitly enabled and available. This keeps the first implementation evaluable and agent-consumable without turning scanner success into a claim about dynamic call resolution or embedding-tool availability.
